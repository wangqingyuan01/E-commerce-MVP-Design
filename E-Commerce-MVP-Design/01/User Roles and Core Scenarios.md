# 用户角色与核心场景 (User Roles and Core Scenarios)

## 1. 模块设计背景

在从0到1构建电商 MVP 系统的初期，明确系统服务的目标用户及其核心诉求是产品架构的基石。为确保研发资源高度聚焦于核心交易流的跑通，本阶段将系统用户严格划分为**需求端（消费者）**与**供给/运营端（商家与平台）**，暂不引入复杂的第三方物流商或分销客等衍生角色。

## 2. 角色定义与场景矩阵

以下为本 MVP 系统支持的核心用户角色及其对应的业务场景边界：

### 2.1 消费者 / 买家 (Customer)
* **角色定位 (Role)**：电商平台的需求方与交易资金的提供方。
* **核心目标 (Core Objective)**：快速发现所需商品，并高效、安全地完成购买与履约追踪。
* **核心业务场景 (Core Scenarios)**：
  * 浏览商品列表与查阅商品详情（图文、价格、库存）。
  * 将意向商品加入购物车，或发起立即购买。
  * 提交订单，确认收货地址，并完成线上资金支付。
  * 在个人中心查看订单流转状态与历史购买记录。
  * 针对异常订单发起取消或退换货（退款）请求。

### 2.2 运营人员 / 商家 (Operations Staff)
* **角色定位 (Role)**：电商平台的供给方与履约服务的执行方。
* **核心目标 (Core Objective)**：维护商品库信息的准确性，管控库存水位，并高效处理用户订单以完成实物交付。
* **核心业务场景 (Core Scenarios)**：
  * 创建新商品并操作上架/下架。
  * 编辑商品基础信息，动态调整 SPU/SKU 价格。
  * 管理 SKU 规格与对应库存状态（防超卖拦截）。
  * 查看并审核用户已支付的订单记录。
  * 录入物流信息，操作订单发货流程（状态扭转）。
  * 协助处理用户的异常订单及退款申请。

---
> **备注 (Product Manager's Note)**：
> 在本 MVP 版本的设计规划中，我们刻意弱化了“平台超级管理员 (Platform Admin)”这一角色。因为在早期验证商业模式阶段，核心痛点是“买卖闭环”而非“内部权限管控”。高级别的权限分配、数据大盘底座等功能将延后至 V2.0 阶段进行系统化建设。
